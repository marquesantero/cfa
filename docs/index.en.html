CFA v2

An architecture for AI-native systems that replaces the static agents-and-skills model with intent resolution, governed planning, validated execution, explicit state projection, and evidence-based lifecycle evolution.

Proposal
From prompt to contract
Instead of asking which agent or skill should act, CFA asks which state change is being requested, under which constraints, and whether it can be executed safely.
Semantic Resolution
Typed intent
Intent becomes a StateSignature with domain, target, datasets, constraints, and execution context.
Governance
Policy before execution
The Policy Engine approves, replans, or blocks before any execution touches real data.
State
Persistent context
The Context Registry closes the loop between what was executed and what may be executed next.
