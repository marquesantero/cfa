CFA v2

Arquitetura para sistemas orientados por IA que substitui o modelo estático de agents e skills por resolução de intenção, planejamento governado, execução validada, projeção explícita de estado e evolução baseada em evidência.

Proposta
Do prompt para contrato
Em vez de perguntar qual agent ou skill deve agir, o CFA pergunta qual mudança de estado está sendo pedida, sob quais restrições e se ela pode ser executada com segurança.
Resolução Semântica
Intenção tipada
A intenção vira uma StateSignature com domínio, alvo, datasets, restrições e contexto de execução.
Governança
Política antes de executar
O Policy Engine aprova, replaneja ou bloqueia antes que qualquer execução toque dados reais.
Estado
Contexto persistente
O Context Registry fecha o loop entre o que foi executado e o que pode ser executado a seguir.
Implementação
Kernel em Python
A implementação atual está organizada em src/cfa, com módulos independentes para resolução, governança, lifecycle e orquestração ponta a ponta.
Módulos
Blocos centrais
resolution, governance, lifecycle e kernel compõem o núcleo operacional do CFA.
Componentes
Fluxo implementado
normalizer, policy, planner, codegen, static_validation, runtime_validation, partial_execution, state_projection, audit e promotion.

Natural language → resolution → confirmation → policy → planning → code generation → static validation → execution → runtime validation → partial execution → state projection → audit → lifecycle
