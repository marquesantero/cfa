# Code of Conduct

## Our Pledge

We want CFA to be an open, rigorous, and respectful collaboration space.

Participants are expected to contribute in ways that improve the technical quality of the project while treating others with respect.

## Our Standards

Examples of behavior that contribute to a positive environment:

- discussing ideas in good faith
- giving technical feedback clearly and respectfully
- accepting critique without escalating to personal attacks
- documenting assumptions and tradeoffs explicitly
- helping keep architecture discussions concrete and reproducible

Examples of unacceptable behavior:

- harassment or discrimination
- personal attacks or insulting language
- repeated bad-faith argumentation
- deliberately derailing technical discussion
- publishing private information without permission

## Scope

This code of conduct applies to:

- issues
- pull requests
- discussions
- commit and review interactions
- other project-managed collaboration spaces

## Enforcement

Project maintainers may remove, edit, or reject contributions and interactions that violate this code of conduct.

Responses may include:

- warning
- temporary restriction
- permanent removal from project spaces

## Reporting

If you experience or witness unacceptable behavior, report it privately to the repository maintainers through the contact channel made available in the repository settings or profile.

## Notes

This document is intentionally concise. The goal is simple:

- high technical rigor
- low interpersonal friction
- zero tolerance for abusive behavior
